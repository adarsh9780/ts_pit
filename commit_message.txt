fix(agent_v3): make planner structured output Azure-strict compatible

Prevents repeated `Invalid schema for response_format 'Plan'` failures on
corporate VDI (Azure OpenAI strict JSON-schema validation).

Changes:
- backend/agent_v3/planning.py
  - switch planner step argument field from dynamic object to string field:
    `tool_args_json`
  - parse `tool_args_json` into dict server-side before building StepState
  - add `json` import and robust fallback on parse errors
- backend/agent_v3/prompts/planner.yaml
  - instruct planner to emit args in `tool_args_json` as valid JSON string

Why:
- Azure strict mode rejects free-form object schemas used in structured output.
- Encoding tool args as JSON string keeps planner schema strict-safe while
  preserving dynamic tool arguments at runtime.
